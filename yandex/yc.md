# Yandex Cloud

Public cloud platform. Resource hierarchy: **Organization → Cloud → Folder →
Resource**. Access permissions inherit parent → child.

Full schema and CLI usage are too large to inline — read these references
when working with Yandex Cloud resources or the `yc` CLI:

- `~/.claude/yandex/refs/yc-reference.md` — platform & `yc` CLI reference
  (IAM, Compute, VPC, S3, Container Registry)
- `~/.claude/yandex/refs/yc-managed-db.md` — managed database services
  (PostgreSQL, ClickHouse, Kafka, Valkey, YDB)
